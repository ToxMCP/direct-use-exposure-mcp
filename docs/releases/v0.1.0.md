# Exposure Scenario MCP v0.1.0

`v0.1.0` is the first public release candidate of Exposure Scenario MCP as the external-dose
construction layer in the ToxMCP suite.

## Included In This Release

- Deterministic dermal, oral, and inhalation screening scenarios
- Tier A uncertainty diagnostics on every scenario output
- Tier B deterministic envelopes and parameter-bounds summaries without probabilistic claims
- Packaged Tier B archetype-library sets for governed envelope construction
- Packaged Tier B archetype-library support for governed Tier 1 personal-care NF/FF inhalation templates
- Packaged Tier C single-driver probability-bounds profiles with curated driver taxonomy
- Packaged Tier C coupled-driver scenario-package probability profiles with curated dependency taxonomy, including governed Tier 1 NF/FF personal-care spray packages
- Machine-actionable Tier 1 inhalation upgrade advisories, `requestedTier` routing hooks, a deterministic NF/FF Tier 1 spray-screening tool, a packaged Tier 1 screening-profile manifest, a dedicated Tier 1 parameter guide resource, and explicit alignment warnings when caller inputs diverge materially from matched Tier 1 profile anchors
- Curated RIVM-backed dermal contact defaults plus product-family spray airborne-fraction defaults for personal-care and household-cleaner screening contexts
- Typed validation dossier publication with benchmark domains, cited external validation datasets, heuristic source families, open evidence gaps, executable validation checks for selected dermal scenarios, and a versioned executable reference-band manifest
- Aggregate/co-use summaries and auditable scenario comparison
- ToxClaw evidence bundles and exposure-refinement bundles with stable hashes
- PBPK handoff export aligned to the published `ingest_external_pbpk_bundle` request shape
- Published contract, release-readiness, security/provenance, and release-metadata resources
- Branch-level defaults curation reporting for curated, route-semantic, and heuristic fallback paths
- Additional adversarial input coverage for control-character rejection, extreme-valid body weights, and strict extra-field handling

## Migration Notes

- This is the first stable public release, so no prior stable migration path applies.
- Any internal pre-release PBPK wrappers that used `toolCall.arguments.bundle` should migrate to
  the published top-level `toolCall.arguments` request payload.

## Known Limitations

- The module remains deterministic-first and does not ship a probabilistic population engine.
- PBPK execution, internal dose estimation, BER derivation, PoD interpretation, and final risk
  decisions remain outside this MCP.
- Remote `streamable-http` deployments still require external authentication and origin controls.
- Some screening defaults remain heuristic and are intentionally surfaced as warning-quality
  assumptions.
- PBPK contract alignment should be re-verified whenever the upstream PBPK MCP changes its
  published ingest request shape.
